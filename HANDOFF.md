# HANDOFF — проект «Анализ дискурса AI-политик»

Памятка для продолжения работы в новом чате/сессии. Проект самодостаточен: всё в этой папке.

> **РАЗВОРОТ (важно).** Главный результат проекта — **дискурсивные типы** AI-политик,
> найденные снизу вверх (`discourse_analysis.py` → `outputs/discourse_*`). Первая версия
> (`main.py`, классификация по дисциплинам) понижена до **вторичной оси**. Если продолжаешь
> работу — отталкивайся от `outputs/discourse_clusters.md` и `outputs/discourse_summary.md`.

## Что это
Учебный проект (курс «Нейронные сети», ИТМО): классификация текстов AI-политик из
университетских силлабусов по дисциплинарным категориям (STEM / Humanities / Professional /
Education_General) + разведочный кластерный анализ. Связан с диссертацией о трансформации
образования в условиях AI.

## Структура
- `discourse_analysis.py` — **главный скрипт** (разворот на дискурс). Грузит кэшированные
  эмбеддинги, пересобирает BERTopic, делает досье кластеров + MLP/SHAP на дискурс-метках.
  Выход: `outputs/discourse_clusters.md`, `discourse_summary.md`, `cluster_assignments.csv`,
  `discourse_{confusion_matrix,learning_curve,shap_summary}.png`. Запуск ~1–2 мин.
- `main.py` — первая версия (дисциплинарная классификация), секции `# === ШАГ N ===`.
  Источник вторичной оси (дисциплина). Запускается целиком, но это уже не главный результат.
- `Syllabi Policies for Generative AI Repository - Policies.csv` — исходные данные (214 строк).
- `outputs/` — все артефакты: `cleaned_policies.csv`, `embeddings_bge.npy`, `embeddings_e5.npy`,
  `labels.npy`, `umap_by_topic.html`, `umap_by_category.html`, `learning_curve.png`,
  `confusion_matrix.png`, `shap_summary.png`, `analysis_summary.md`, `requirements.txt`.

## Ключевые решения (чтобы не переоткрывать)
- **Эмбеддинги**: сравниваются две модели — `BAAI/bge-base-en-v1.5` (англ., основная по идее)
  и `intfloat/multilingual-e5-large` (мультиязычная, задел под русский корпус). Обе MIT.
  По факту победила e5 (acc 0.524 vs 0.429), она выбрана автоматически как лучший бэкенд.
- **Категории**: маппинг дисциплин по ключевым словам/подстроке (не точное сравнение строк),
  остаток `Other` сворачивается в `Education_General`.
- **Классификатор**: sklearn `MLPClassifier(256,128,64)` + honest baseline LogisticRegression.
- **SHAP**: на отдельном TF-IDF MLP (на 768/1024-мерных эмбеддингах SHAP по словам не читается).
- Слово «writing» доминирует в SHAP по всем классам — это НЕ баг, а авторская форма корпуса
  (38 курсов Writing). Намеренно НЕ вычищается.

## Текущие результаты (последний прогон)
- 208 документов; классы: Humanities 75 / Education_General 52 / Professional 49 / STEM 32.
- BERTopic: 14 кластеров, 23.6% шума.
- MLP accuracy (e5): 52.4% (выше baseline ~36% и доли мажорного класса ~36%).

## ЧТО ОСТАЛОСЬ СДЕЛАТЬ (за автором, не за кодом)
Главное — интерпретация дискурс-кластеров в `outputs/discourse_summary.md`:
1. **§6** — прочитать `discourse_clusters.md` и дать название каждому укрупнённому кластеру.
2. **§7** — соотнести с типологией диссертации (минимализм / гуманистический / технократический).
3. **§8** — выводы.
(В старом `analysis_summary.md` тоже есть незаполненные §6/§8 — но это вторичная, дисциплинарная ветка.)
